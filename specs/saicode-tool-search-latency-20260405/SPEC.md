# saicode tool/search latency Spec

## Goal

对 `saicode` 当前“会调用本地工具 / 会做 WebSearch”的真实非交互链路做一次前台测速，判断体感是否达标；若发现明显偏慢且能安全收口的热点，同轮直接修复并复测。

## Scope

### In scope

- 真实运行以下类型探针并记录结果：
  - 本地读取类工具
  - 本地搜索类工具
  - WebSearch
- 记录至少以下信息：
  - end-to-end wall time
  - RSS
  - 是否真实命中预期工具/搜索链路
  - 结果是否正确
- 若发现明显偏慢的单点：
  - 定位具体慢在启动、工具装载、还是搜索实现
  - 在当前仓库内做低风险修复
  - 复测前后差异

### Out of scope

- 全面重写搜索系统
- 重做 provider / model 路由
- 为了测速而更改生产模型策略

## Constraints

- 必须以真实 `./bin/saicode -p ...` 探针为准，不只看代码推测
- 不破坏现有工具/搜索正确性
- 不能回退用户已有脏工作区改动
- 只修“证据明确且收益清楚”的慢点，不做无关扩张

## Acceptance

1. 至少完成一组真实本地工具探针与一组真实 WebSearch 探针。
2. 每组都给出 wall time、RSS、结果正确性与体感评价。
3. 若发现明显慢点，给出根因判断并完成一次修复复测；若未发现足够明确的热点，也要明确说明。
4. 自动化验证与最小真实复测通过后再交付。
