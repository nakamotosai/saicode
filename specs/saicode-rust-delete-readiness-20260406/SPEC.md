# saicode Rust delete-readiness spec

日期：
- 2026-04-06

目标：
- 以真实运行验证判断当前 saicode Rust 主链是否已经达到“可删除 TS/Bun 旧版”的程度。

范围：
- 核对 TS 代码是否只是备份，还是仍有活跃回退路径。
- 实测 Rust 默认入口是否可正常执行常见 prompt。
- 实测 Rust 版是否能识别明确工具意图并正确调用工具。
- 实测 Rust 版是否能完成“给可用模型做 TTFT 测速”这类高频代理任务。

非目标：
- 本轮不做大规模 Rust 迁移。
- 本轮不先入为主删除任何 TS/Bun 旧代码。

约束：
- 结论必须基于真实命令与真实模型调用，不以文档阶段名代替验收。
- 若命中失败，必须区分是 Rust 主链能力缺口、工具权限问题、还是测速任务设计问题。

验收：
- 能明确回答 TS 51 万行是否仍有活跃运行价值。
- 能给出 Rust 主链的通过项与失败项。
- 能明确判断“现在可删 TS/Bun”或“现在还不能删”，并列出阻塞点。
